# ЛР6 — Висновки

> **Термінологія:**  
> **RFM** (Recency, Frequency, Monetary) — модель сегментації за давністю / частотою / сумою покупок · **ABC** — поділ клієнтів за внеском у дохід (принцип Парето 80/15/5%) · **XYZ** — поділ за стабільністю поведінки · **COGS** — прямі операційні витрати ($0.005/подія) · **JTBD** (Jobs To Be Done) — методологія визначення «завдання», заради якого користувач прийшов у продукт · **VIP** — сегмент найцінніших клієнтів · **SLA** — рівень якості сервісу · **API** — міжсервісна взаємодія

## 1.1 RFM-сегментація та скоринг

| Сегмент | Користувачів | % доходу |
|---------|-------------|----------|
| Champions (555) | 49 | 2.57% |
| At Risk (211–255) | 167 | 3.16% |
| Hibernating (111) | 36 | 0.11% |

**Інтерпретація:** Сегмент At Risk дає вищу частку доходу, ніж Champions (3.16% проти 2.57%). Це пояснюється тим, що Champions визначається через строгу рівність балів «555», тоді як At Risk охоплює ширший числовий діапазон. На практиці: для Champions — програма утримання VIP-рівня та пріоритетна підтримка; для At Risk — win-back кампанія є найвищим бізнес-пріоритетом, оскільки ці клієнти вже давали значний дохід.

## 1.2 ABC/XYZ-матриця та перетин з RFM

- Одночасно в сегменті **555 та категорії A**: **45 користувачів**
- **Технічна пріоритизація для класу A:** виділений ліміт запитів в API (без throttling), підвищений SLA відновлення після збою, участь у beta-тестуванні фіч, окремий кеш-шар.

Розподіл ABC/XYZ:
- A/X: 2 users, дохід 631
- A/Y: 131 users, дохід 26 702
- A/Z: 1 757 users, дохід 262 270
- B/Y: 19 users, дохід 970
- B/Z: 1 158 users, дохід 52 177
- C/Y: 3 users, дохід 66
- C/Z: 1 349 users, дохід 19 349

Домінуюча комбінація — A/Z: велика кількість клієнтів з високим доходом, але нерегулярною активністю. Це типово для e-commerce без підписочної моделі.

## 2.1 Маржинальний аналіз (витрати $0.005 / подія)

| Метрика | Значення |
|---------|----------|
| Користувачів з від'ємною маржею | 266 088 (98.49%) |
| Середня валова маржа | 1.2611 |
| Мінімальна маржа | -5.44 |
| Максимальна маржа | 1526.99 |

**Стратегія Rate Limiting для збиткових користувачів:**
- Ліміт на кількість подій: квота на рівні ~500 events/day для профілів з від'ємною маржею
- Агресивне кешування для «важких» переглядачів без конверсії
- Throttling подій `view_item` для сегментів з margin < 0 протягом 7-денного вікна
- Перехід на асинхронний batch-запис некритичних аналітичних подій замість синхронного

## 2.2 JTBD-сегментація за поведінковими паттернами

| Паттерн | Користувачів | % |
|---------|-------------|---|
| Price Comparison | 3 822 | 32.15% |
| Window Shopping | 8 066 | 67.85% |

**Фіча для Price Comparison:** оповіщення про зниження ціни («ціна впала на X% від вашого перегляду»), бейдж «найнижча ціна за 30 днів», інструмент порівняння товарів у два кліки — зменшить friction між browse і add_to_cart.

**Window Shopping:** персоналізовані добірки на основі переглянутих категорій, обмежені за часом пропозиції з лічильником, сегментований email-retargeting.
